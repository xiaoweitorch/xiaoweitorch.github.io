Knapsack for All Subsets

编号：0026

试题来源：AtCoder

[TOC]

试题描述

给定有$N$个正整数的一个序列$A_1,\cdots,A_N$，以及一个正整数$S$。

对于任意一个$\{1,2,\cdots,N\}$的非空子集$T$，定义$f(T)$如下：

  • $f(T)$是互不相同的$T$的非空子集$x_1,x_2,\cdots,x_k$满足$A_{x_1}+\cdots+A_{x_k} = S$

请找出$\{1,2,\cdots,N\}$的所有非空子集$T$的$f(T)$的和，该值模998244353为结果。

限制条件：

  • 所有的输入是整数
  • $1\leq N\leq 3000$
  • $1\leq S\leq 3000$
  • $1\leq A_i\leq 3000$

输入

N S

$A_1$ $\cdots$ $A_N$

解答算法

首先，假定对于满足$A_{x_1}+A_{x_2}+\cdots + A_{x_k}=S$的这样的子序列，那么显然，包含这个子序列的序列数目就为$2^{N-k}$，因此对于每个序列，其每多包含一个数字，包含这个序列的序列的可能性就会是原来的$\frac{1}{2}$。

因此我们可以设置一个$dp[s]$，$dp[s]$代表当前满足$A_{x_1}+A_{x_2}+\cdots + A_{x_k}=S$的个数。

初始的时候有$dp[0]=2^N$，这是显然的。

然后我们观察递推方程，假设当前读入了数字$x$，那么显然对于任意的$j\geq x$，都有$dp[j] = dp[j] + dp[j-x]/2$，为什么后面的$dp[j-x]$要除以二呢，原因就是你多加了一个数字$x$进入序列，那么其可能数目变为原来的$\frac{1}{2}$。

这个题目因为用到了求模，所以说在求模中的除法和正常不一样，要用到费马小定理，也就是把$(a/2)mod\quad p$转化成$a*2^{mod-2}$

但是这个题目里面，看厉害的大佬解答，都是用$(a/2)mod \quad p$转换成了$a*(mod-mod/2)$,这个数学原理是什么不太了解，但是正确性可以简单的想一下，两个$a*(mod-mod/2)$相加，一定为$1$，因此一个是原来的$\frac{1}{2}$，如果有谁知道原理，希望指点我一下。

Count Median

编号：0025

题目来源：AtCoder

[TOC]

题目描述

假定有$N$个整数$X_1,X_2,\dotso,X_N$，我们知道其中的任意一个整数$A_i\leq X_i \leq B_i$，也就是说我们知道每一个数取值的上下界，要给出这$N$个整数的中位数的可能取值情况。

当$N \& 1 = 1$的时候，也就是$N$是奇数的时候，中位数是$x_{\frac{N+1}{2}}$

当$N\&1=0$的时候，也就是$N$是偶数的时候，中位数是$(x_{\frac{N}{2}} + x_{\frac{N}{2} + 1})/2$

取值范围：

  • $2\leq N\leq 2\times 10^5$
  • $1\leq A_i \leq B_i\leq 10^9$
  • 所有输入均为整数

输入

N

$A_1$ $B_1$

$\vdots$

$A_N$ $B_N$

解答算法

算法思路

很显然，如果一个数$x$可以做数列$X_1,\cdots,X_N$的中位数，那么起码要有一半的数字可以比$x$小，那么让所有的数字都取到他们的最小值，然后该数列$A_1,\cdots,A_N$的中位数就应该是整体中位数的最小值；同时也要有一般的数字能够比$x$大，因此，让所有的数字都取到他们的最大值，然后该数列$B_1,\cdots,B_N$的中位数就是整体中位数的最大值。

当$N$为奇数的时候，那么中位数的最小变动是1，因此$[a,b]$中会有$b-a+1$个可能取值。

当$N$为偶数的时候，那么中位数的最小变动是$0.5$，因此$[a,b]$中会有$2*(b-a)+1$个可能取值。又因为当$N$是偶数的时候，a和b都是两个数相加的和，因此可以看作$(b_1+b_2)-(a_1+a_2)+1$

对于求解数列的中位数，比较简单的方法就是直接进行排序，然后去下标中间的数字，这样做的时间复杂度是$O(nlogn)$，因为排序的时间复杂度是$O(nlogn)$

也可以采用最大最小栈的方法，设定两个优先栈$max,min$，然后每一次输入新的数字，都对两个栈进行替换，这样因为只输入了$n$个数字，最终时间复杂度应为$O(n)$

代码实现

  • 排序实现

复杂度分析

  • 时间复杂度：一个是$O(nlogn)$，一个是$O(n)$
  • 空间复杂度：都为$O(n)$

Div Game

编号：0024

试题来源：AtCoder

题目描述

给定一个正整数$N$，思考对$N$进行下面重复的操作

  • 首先，选定一个正整数$z$满足下面所有的要求：
    • $z$能够被表示为$z=p^e$，这里面的$p$是质数，$e$是正整数
    • $N\%z ==0$
    • $z$和先前选择的整数不同
  • 然后继续对$N/z$进行上述操作

找到这个操作能够进行的最大次数

条件

  • 所有的输入都是整数
  • $1\leq N\leq 10^{12}$

输入

N

解答算法

算法思路

题目中要求$z = p^e$，而且$p$又是质数，然后$z$还要求是原来数字$N$的一个因数，很显然这道题目相当于质因数分解的题目。

假设我们已经解决了质因数分解的问题，已经得到了数字$N$分解出的质因数，那么我们会有一个map，对于每个数字$N$的质因数，对应的值是其出现的次数，例如$8$的质因数为(<2,3>)，表明2作为其质因数，出现了3次。

要想弄清楚操作的次数，对于任意的一个质因数$$，最开始，可以选取$i^1$作为$z$，然后可以选取$i^2$，依次增大，一直到$1+2+\dots +j\leq k 且1+2+\dots+j+(j+1)>k$，那么$j$就是对于质因数$$所能进行的最多操作次数。要求整体的最多操作次数，只要求$\sum\limits_{map.begin()}^{map.size}j$。

因此这道题目的关键在于求解质因数。

求解质因数可以采用递归的思路进行处理。首先设置工具函数is_prime来检验数$i$是否为质数。然后对于一个数$n$，求解其质因数，从2开始进行遍历，遍历到$n$为止，判断其中是否存在一个数$i$是质数，同时满足$n\%i==0$。找到这样的数，循环终止。对$n/i$调用上述过程。如果$n/i$是质数，递归终止。

代码实现

复杂度分析

  • 时间复杂度：$O(n^2)$
  • 空间复杂度：$O(n)$

Multiplication 2

编号：0023

试题来源：AtCoder

试题描述

给定$N$个整数$A_1,\dotso,A_N$，计算$A_1\times \dots\times A_N$。

当乘积的结果大于$10^{18}$，输出-1而不是乘积

说明：

  • $2\leq N\leq 10^5$
  • $0\leq A_i \leq 10^{18}$
  • 所有的输入都是非负整数

输入形式

N

$A_1$ $\dots$ $A_N$

解答算法

算法思路

首先如果输入中存在$A_i=0$，那么最后乘积的结果一定是0，因此首先在读入数据的过程中判断是否存在$A_i=0$。

若不存在$A_i=0$，那么要知道该数组的乘积是个非递减数列，因为每个数$A_i\geq 1$，因此如果计算到$A_j$的时候，当前的result已经大于$10^{18}$，那么继续乘下去，也是绝对会大于$10^{18}$。

关键点在于，result不一定会在unsigned long long的取值范围中，可能存在溢出的问题，当溢出的时候，显然一定也是大于$10^{18}$，因此注意考虑溢出的情况。

这里用到了一个很巧妙的比较

只需要进行乘积的对比就可以了，eps是一个极小数，取得值是$10^{-6}$，是double正常的误差数，这样如果超出的话，显然不满足大于eps的条件，巧妙地避开了溢出问题

  • 时间复杂度：$O(n)$，整个过程对数组进行了两次遍历
  • 空间复杂度：$O(n)$

0022 柱状图中最大的矩形

发表于 2020-05-30 更新于 2020-06-09 分类于 每天一道算法题 Disqus：
本文字数： 1.7k 阅读时长 ≈ 2 分钟
柱状图中最大的矩形

编号：0022

试题来源：leetcode

[TOC]

题目描述

给定$n$个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为1。

求在该柱形图中，能够勾勒出来的矩形的最大面积。

解答算法

算法思路

我们在思考最大的矩形的时候，可以任意选取一个柱子i，以其高度height[i]作为基准，向左右两侧进行拓展，如果拓展到的柱子的高度大于等于height[i]，那么其可以作为以当前高度作为高的举行的一部分，如果小于，那么就可以得到其边界。

这样可以得到左右边界，有了左右边界，同时又有高度，就有了面积。

要得到左右边界，最笨的方法是对其进行两次遍历，每一次都要从当前位置i进行向左向右遍历，显然最坏的时间复杂度为$O(n)$。

我们可以使用一个栈stack存放单调的heigth[i]。考虑这样一个事实，如果有$height[i]\geq height[j]且i<j$，那么显然从大于$i$的一个位置进行寻找边界，无论如何也不能选取$i$作为左边界，因为$j$挡住了$i$，因此可以设置这样一个单调栈stack，存放height[i]，以单调递增的顺序存放。

当我们处理位置i的时候，假设栈中已经存在$a_0,a_1,\dots ,a_k,a_{k+1},\dots,a_m$，那么假设$height[i] > a_k$，那么显然$a_{k+1},\dots,a_m$都不能阻挡i的向左拓展，因此$k$就是其左边界。要找到左边界，我们只需要将栈顶大于等于height[i]的值都pop出来，然后当前栈顶的坐标就是左边界。得到左边界后再将$heigth[i]$加入栈中，形成新的单调栈

代码实现

复杂度分析

  • 时间复杂度：整个过程中对数组只进行一次遍历，可以看出在单调栈的操作过程中，我们对每个元素只进行了一次入栈操作，同时也最多只有一次出栈操作，因此时间复杂度为$O(n)$
  • 空间复杂度：申请了左右边界数组，长度都为$n$，因此空间复杂度为$O(n)$

  • $1\leq A.length\leq 30000$

逐一统计

算法思路

代码实现

复杂度分析

  • 时间复杂度：$O(n)$，整个过程中只对数组A进行了一遍遍历
  • 空间复杂度：$O(min(m,n))$，其中n是A的长度，m是K的大小
